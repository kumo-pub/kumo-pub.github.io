# 经典模式

在经典模式下使用 kmpkg 来共享已安装库的中央实例。

kmpkg有两种运行模式：经典模式和
[清单模式](manifest-mode.mdx)。对于大多数用户，我们建议使用清单模式。

在经典模式下，kmpkg 在 kmpkg 内维护一个中央“安装树”
由单独的 [`kmpkg install`](../commands/install.mdx) 构建的实例和
[`kmpkg 删除`](../commands/remove.mdx) 命令。这组核心包
然后可以由任意数量的项目共享。

因为已安装的软件包集与单个软件包没有关联项目中，经典模式的操作类似于`brew`或`apt`等工具。然而，
该集合仍然与 kmpkg 实例关联，并且 kmpkg 的每个实例
（比如单独的`git clone`副本）会有自己的一套经典模式
安装的软件包。

要了解如何在经典模式下安装依赖项，请参阅[教程：从命令行安装依赖项](../consume/classic-mode.mdx)。
