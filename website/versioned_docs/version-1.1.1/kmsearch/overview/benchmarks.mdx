# kumosearch 性能测试

以下是 kumosearch 在不同数据集上的性能测试结果：

- 数据集1：**220 万食谱**（包含食谱名称和成分）:
  - 索引时占用约 900MB 的 RAM
  - 索引 220 万条记录耗时 3.6 分钟
  - 在 4 vCPU 的服务器上，kumosearch 能够处理**每秒 104 个并发搜索查询**，平均搜索处理时间为 **11 毫秒**

- 数据集2：**2800 万本书**（包含书名、作者和类别）:
  - 索引时占用约 14GB RAM
  - 索引 2800 万条记录耗时 78 分钟
  - 在 4 vCPU 的服务器上，kumosearch 能够处理**每秒 46 个并发搜索查询**，平均搜索处理时间为 **28 毫秒**

- 数据集3：**300 万产品**（Amazon 产品数据）
  - 在 8 vCPU 的三节点高可用 kumosearch 集群上，kumosearch 能够处理**每秒 250 个并发搜索查询**的吞吐量
