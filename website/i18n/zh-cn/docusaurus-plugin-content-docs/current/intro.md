# AI 时代的 C++ 开发操作系统

让个人开发者 1 人 2 天上线生产级分布式系统。

## 行业背景：繁荣背后的`基建危机`

AI 浪潮正以前所未有的速度重塑世界。然而，当 AI 步入商业化的 **“深水区”** ——即需要支撑百亿级数据量、毫秒级响应、
复杂的在线排序、广告检索与智能推荐，底层的 C++ 工程体系却暴露出了两个致命痛点：

- **碎片化的底层基建（The Infrastructure Chaos）**：
    C++ 生态长期缺乏统一的包管理与工程标准。依赖冲突、多平台适配、复杂的编译选项，让 80% 的开发精力耗费在“修水管”而非业务创新。

- **工程落地的巨大鸿沟（The Production Gap）**：从实验室的算法模型到工业级的生产环境，存在极高的门槛。高性能搜索、分布式调度、
  列式存储、列式计算等大厂级能力，对于中小型企业而言几乎是“技术壁垒”。

Kumo 的目标是成为 AI 时代的 **C++ 开发的“操作系统”**，让中小公司，甚至个人开发者，
**在 2 周内拥有大厂级别的基建能力，让技术能力不再成为业务创新的壁垒**

## Kumo 的核心价值：平权技术，效率革命

Kumo 的核心使命是**平权技术**，释放业务创新与产品迭代的潜力：

- **极速落地**：让中小公司甚至个人开发者，在 2 周内即可搭建起大厂级别的搜索、推荐与 AI 基础设施。
- **工程下沉**：通过一套工具链，搞定 C++ 依赖、编译、部署、环境一致性，实现无入侵的工程管理，彻底解决依赖与构建难题。
- **效率革命**：将原本需要多部门协同、耗时数周的业务迭代，缩短至天级开发、秒级生效。


## Kumo 能为你做什么？

Kumo 通过 **“双轮驱动”** 模式，同时解决工程基建的**深度**与业务落地的**速度**。

- 一轮：C++ 工程基建（解决编译 / 依赖 / 构建）
- 二轮：AI 搜索 / 推荐 / RAG（检索增强生成）业务能力（直接上线可用）

> 实战佐证：基于该双轮驱动模式，我们已在 [ksearch](https://github.com/kumose/ksearch) 项目中实现**1人仅用2天开发并上线基于 Raft 集群的生产级 KNS 功能**，验证了基座支撑下“天级开发、生产级可用”的核心能力。

### C++ 工业级开发基座：彻底终结“修水管”时代

- **统一依赖管理**：通过 `kmpkg` 一键集成 300+ 核心 C++ 组件（包含高性能 RPC、存储引擎、内存布局等），告别漫长的源码编译与版本冲突困扰。
- **工程标准化下沉**：利用 `kmdo` 和 `kmcmake` 实现 CI/CD 无缝集成。让项目管理意志跨越平台与环境限制，实现真正的“代码即环境”。
- **顶级基建整合**：深度整合 Arrow、Velox、Faiss、brpc 等顶级开源能力，提供统一的数据模型与内存调度系统。让你的技术起跑线，直接对标大厂的演进终点。

### 业务“最后一公里”落地：让 AI 场景秒级生效

- **AI 搜索与推荐平权**：将支持百亿级流量的向量搜索、混合搜索能力产品化与 API 化。让中小团队也能轻松驾驭大模型时代的检索增强（RAG）与复杂推荐场景。
- **SQL IR（中间表示）效率革命**：通过 SQL/CTE 定义业务规则，将复杂的排序逻辑与 AB Test 转化为标准算子插件化。原本需要多部门协同、耗时数周的改动，现在仅需 **“天级开发、秒级上线”**。
- **全链路工业化闭环**：从数据模型定义到分布式集群调度，Kumo 补齐了算法到生产环境的最后一块拼图，让算法逻辑像工业流水线一样精准、高效地运转。
